## Design System (Locked)

### Colors
- `--color-brand-blue: #00469C` (accent, links, active nav)
- `--color-amber: #C4883A` (role labels, section underlines, carousel buttons)
- `--color-charcoal: #1c1c19` (primary text)
- `--color-bg: #fcf9f4` (page background — warm off-white)
- `--color-surface: #f0ede8` (carousel panel background)
- `--color-text-muted: #434752` (body text, descriptions)

### Typography
- **Headlines:** Plus Jakarta Sans (weight 700–800)
- **Body:** Manrope (weight 400–600)
- Hero headline: `clamp(2.25rem, 6vw, 4.25rem)` — reduced from original to prevent "multi-camera" wrapping
- Brand titles in carousel: `clamp(1.75rem, 4vw, 2.75rem)`, weight 800
- Subtitles: `clamp(0.95rem, 1.8vw, 1.1875rem)`, weight 700, muted color
- Role labels: 0.875rem, uppercase, amber color, `border-bottom: 2px solid amber`, `align-self: flex-start`

### Layout
- Mobile-first with breakpoints at 768px and 1024px
- Max width: 80rem (1280px)
- Carousel content: single-column stacked, max-width 56rem, media on top
- Media containers: 16:9 aspect ratio, rounded corners, shadow

## How the Carousel Works

`js/carousel.js` fetches `data/projects.json` at runtime and renders one project card at a time with fade transitions (180ms out, 280ms in).

### Card structure (rendered by JS):
1. **Media** — 16:9 container with either `<img>` or `<video autoplay muted loop playsinline>`
2. **Brand** — large headline (e.g., "Magic: The Gathering")
3. **Subtitle** — optional, smaller (e.g., "Pro Tour")
4. **Role** — amber uppercase label with underline (e.g., "ASSISTANT DIRECTOR")
5. **Description** — one or more `<p>` tags. The JSON uses `\n\n` as a paragraph delimiter, which `renderDescription()` splits into separate `<p>` elements.

### Title system:
- Some cards have brand + subtitle (e.g., "Magic: The Gathering" / "Pro Tour")
- Some cards are brand-only (e.g., "Fanatics Fest NYC", "Crunchyroll Anime Awards")
- The subtitle is optional in the JSON schema

### projects.json entry schema:
```json
{
  "id": "kebab-case-id",
  "brand": "Brand Name",
  "subtitle": "Optional Subtitle",
  "role": "Role Title",
  "description": "First paragraph.\n\nSecond paragraph.",
  "media": {
    "type": "image | video",
    "src": "assets/projects/filename.ext",
    "alt": "Accessible description.",
    "poster": "optional-poster.jpg"
  }
}
```

### JSONP fallback:
When opened via `file://` protocol (local preview), `fetch()` fails due to CORS. The carousel catches the error and falls back to loading `data/projects.js` via `<script>` injection. That file is a JSONP wrapper:

```
__loadProjects({ "projects": [ ... ] });
```

**After any change to `projects.json`, regenerate `projects.js`:**
```bash
echo -n "__loadProjects(" > data/projects.js && cat data/projects.json >> data/projects.js && echo ");" >> data/projects.js
```

## Known Issues & Unresolved Items

### Open decision: UW Building Community role line
Richard asked whether the subtitle "Building Community Through Live Streaming" should include "graduate course." The description already opens with "A graduate course I designed..." but the subtitle doesn't convey the academic weight. The recommended approach (not yet implemented) is to change the **role line** from "Course Creator & Instructor" to **"Graduate Course Creator & Instructor"** rather than making the subtitle longer. Awaiting Richard's approval.

### Description below the fold on desktop
On desktop, the 16:9 media at the top of each carousel card takes up significant viewport height, pushing the description below the fold. A two-column layout (media left, info right) was attempted and reverted — Richard was skeptical it would help enough. This remains an unresolved UX concern. Don't attempt to solve it without discussing with Richard first.

### projects.js sync
`data/projects.js` must be regenerated whenever `data/projects.json` changes. It's easy to forget. Always regenerate after edits.

### README is slightly outdated
The README still uses the old `title` field name instead of `brand`/`subtitle` in its JSON examples. Should be updated to match the current schema.
